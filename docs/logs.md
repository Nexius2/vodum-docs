The **Logs** page provides a detailed view of **all application logs generated by VODUM**.

Logs are essential for:
- monitoring system activity
- understanding task execution
- debugging issues
- auditing automated operations

## ğŸ” Log filters

At the top of the page, several tools allow you to filter logs efficiently.

### Log level filter
You can filter logs by severity level.

Common levels include:
- **INFO**: normal operations and successful actions
- **WARNING**: non-critical issues
- **ERROR**: failures or unexpected behavior
- **DEBUG** (if enabled): detailed diagnostic information

> ğŸ’¡ In production, INFO is usually sufficient.

---

### Search
The **Search** field allows full-text search across:
- messages
- sources
- task names
- identifiers

Search is applied in real time.

## â±ï¸ Task-related logs

Many logs are generated by **automated tasks**.

Typical task log flow:
1. Task scheduled or delayed
2. Task start
3. Task internal processing
4. Task completion (success or failure)
5. Next run scheduling

Example messages:
- `TASK START`
- `TASK SUCCESS`
- `No job to process`
- `Next run at ...`

> ğŸ’¡ These logs are extremely useful when diagnosing automation issues.

## ğŸ§¯ Troubleshooting with logs

### Tasks not behaving as expected
- filter logs by task name
- look for START / SUCCESS / ERROR entries
- check next run scheduling

---

### Emails not sent
- search for `send_mail`
- verify SMTP-related errors
- confirm mailing tasks execution

---

### Access not updated
- search for `apply_plex_access_updates` or `apply_jellyfin_access_updates`
- look for skipped or failed jobs

---

### Missing logs
- verify **Debug mode** if detailed logs are required
- ensure logging is enabled at system level
